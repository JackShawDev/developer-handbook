# Developer Handbook

A personal knowledge base documenting my journey towards becoming a well-rounded IT professional.

This handbook contains my own notes, explanations and practical experience gained through learning, experimentation and building software. It is intended to be a long-term reference that grows alongside my skills and is written using official terminology wherever possible.

---

## Objectives

The goals of this handbook are to:

- Build a strong understanding of software engineering fundamentals
- Create a personal reference for concepts and tools
- Develop good engineering habits and documentation practices
- Track my progress
- Support future interview preparation and professional development

## Learning Principles

Throughout this handbook I aim to follow a small number of principles:

- Understand concepts before memorising syntax
- Use official terminology whenever possible
- Learn by building, experimenting and reflecting
- Keep documentation concise, accurate and maintainable
